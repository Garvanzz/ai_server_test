# MySQL 表结构管理（方式 A + B，无需代码）

## 思路

- **唯一真相**：`000_schema.sql` 表示当前版本的**完整建表**，改字段时优先改这一份。
- **方式 B（清空后重建）**：执行 `clear.sql` 清空库，再执行 `000_schema.sql` 建出所有表。
- **方式 A（已有表改字段）**：在 `000_schema.sql` 里改定义，并新增一条**增量迁移**（如 `002_xxx.sql`）写 `ALTER TABLE`，在已有库上按顺序执行迁移即可统一改表。
- **更好做法**：日常改表只加迁移、不改历史迁移内容；`000_schema.sql` 在发版或“全新环境”时与迁移结果对齐，便于以后再次“清空 + 建表”得到最新结构。

## 目录约定

```
scripts/mysql/
  000_schema.sql   # 完整建表（唯一真相，用于清空后全量建表）
  clear.sql        # 清空：删除所有业务表（执行后再执行 000_schema.sql 即“清空并建表”）
  001_xxx.sql      # 增量迁移（按顺序执行，用于已有库的改字段/加表）
  002_yyy.sql
  ...
  README.md        # 本说明
```

## 使用方式

### 清空当前库并创建所有表（方式 B）

1. 确认当前库可清空（备份如有需要）。
2. 执行：`mysql -u用户 -p 库名 < scripts/mysql/clear.sql`
3. 执行：`mysql -u用户 -p 库名 < scripts/mysql/000_schema.sql`

### 已有表要改字段（方式 A）

1. 在 **000_schema.sql** 里改对应表的定义（保证“唯一真相”是最新的）。
2. 新增一条迁移文件，如 `002_add_foo_to_account.sql`，内容示例：
   ```sql
   -- 002_add_foo_to_account.sql
   ALTER TABLE account ADD COLUMN foo VARCHAR(64) NOT NULL DEFAULT '' COMMENT 'xxx';
   ```
3. 在**每个需要更新的环境**上，只执行**尚未执行过的**迁移（按编号顺序）。  
   （执行记录可自建一张表或由迁移工具维护；此处不提供代码，可用手工或任意迁移工具。）

### 全新环境 / 新服

直接执行 `000_schema.sql` 即可（无需先 clear，库是空的）。若该库曾用迁移迭代过，也可先 clear 再执行 `000_schema.sql`，得到与当前“唯一真相”一致的结构。

## 注意

- 生产环境改表建议只用**增量迁移**（ALTER），不要在生产上直接“清空 + 建表”。
- `000_schema.sql` 与迁移结果不一致时，以迁移为准；发版前可把当前库结构导出一份，反写回 `000_schema.sql`，便于下次“清空 + 建表”仍是最新结构。
