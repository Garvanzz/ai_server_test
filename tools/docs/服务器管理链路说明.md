# 服务器管理链路说明

## 1. 整体架构

```
Admin 前端 (Vue)
    │  HTTPS/HTTP POST
    ▼
gm_server（GM 管理后端）
    │  TCP 内网
    ├─ 查询/写入 gm_db（管理数据库）
    │    ├─ game_server 表 —— 区服基本信息、MainServerHttpUrl
    │    └─ server_process 表 —— 进程管理记录（启动命令、健康检查 URL 等）
    │
    └─ HTTP 内网
         ▼
       main_server（大厅服）
           ├─ GET  /ping         ← 健康检查（新增）
           └─ POST /gm/*         ← GM 操作转发（邮件、踢人、背包等）
```

---

## 2. 运行状态判断逻辑

### 2.1 区服列表（大厅服）— `运行状态` 字段

**调用链路：**  
`Admin GET /api/servers/game-list`  
→ `gm_server GmGetGameServerList`  
→ 读取 `game_server` 表（`group_id > 0`）  
→ `buildManagedServerItem()` → `managedServerReachable()` → `mainServerHTTPReachable(item.MainServerHttpUrl)`

**判断方式（修复后）：**  
```
HTTP GET {MainServerHttpUrl}/ping
  响应 200 → "运行中"
  超时/连接拒绝/非200 → "离线"
  MainServerHttpUrl 为空 → "离线"
```

超时时间：**800ms**

> **旧逻辑（已修复）**：只做 TCP `DialTimeout`，只要端口监听即显示"运行中"，即使 HTTP 层崩溃也不感知。  
> **新逻辑**：发起真实 HTTP GET 到 `/ping`，1端口+HTTP 层同时存活才算"运行中"。

### 2.2 进程管理 — `运行状态` 字段

**调用链路：**  
`Admin GET /api/servers/game-list-all` / `/api/processes/list`  
→ `gm_server GmGetGameServerProcessList` / `GmListProcesses`  
→ 读取 `server_process` 表  
→ `buildProcessRespItem()` → `processRunState()` → `processHealthCheck()`

**判断方式（两级降级）：**

| 优先级 | 条件 | 检测方式 |
|--------|------|---------|
| 1 | `http_health_url` 非空 | TCP `DialTimeout(host, 800ms)` |
| 2 | `process_bin_name` 非空 | 查询系统进程名（见下表）|
| 兜底 | 两者均为空 | 始终返回"离线" |

**进程名查询命令：**

| 平台 | 命令 |
|------|------|
| Linux | `pgrep -x <name>` |
| Windows | `tasklist /FI "IMAGENAME eq <name>" /FO CSV /NH` |

## 3. 进程管理（启动/停止/重启）流程

### 3.1 大厅服（main_server）操作

**Admin 按钮 → API 映射：**  
`POST /api/servers/start` → `GmStartServer` → `handleManagedServerLifecycle`

**请求体：** `{ "serverId": <game_server.id> }`

**后端逻辑：**
1. 以 `server_ref_id = serverId` 查询 `server_process` 表（一个区服可有多条进程记录）
2. 遍历所有关联进程记录，依次执行 `startProcess()`
3. 要求 `manage_mode = "local_command"` 且 `start_command` 非空

**启动进程（跨平台）：**

| 平台 | 实际执行 |
|------|---------|
| Linux | `sh -c <start_command>` |
| Windows | `cmd /C <start_command>` |

进程启动后**不等待**，后台异步记录退出日志。

**停止进程：**

| 平台 | 命令 |
|------|-----|
| Linux | `pkill -x <process_bin_name>` |
| Windows | `taskkill /IM <process_bin_name> /F` |

### 3.2 游戏服（game_server / battle_server）操作

**Admin 按钮 → API 映射：**  
`POST /api/servers/game-start` → `GmStartGameServer` → `handleProcessLifecycle`

**请求体：** `{ "id": <server_process.id> }`  

> **注意**：此处是 `server_process.id`（进程记录主键），而非 `game_server.id`。  
> 前端 `handleStart(row, 'game')` 传递的是 `row.gameServerId`，该字段由 `buildDashboardRows()` 赋值为 `game.id`（即 server_process.id）。

**后端逻辑**与大厅服相同，单条进程记录操作。

## 4. 数据库关联关系

```
game_server 表（区服）
  id              → 区服唯一ID（也是 GmGetGameServerList 返回的 entry.id）
  logic_server_id → 逻辑服编号（客户端选服用）
  main_server_http_url → 大厅服 HTTP 地址，用于健康检查
  group_id        → 关联 server_group（区服组/合服组）

server_process 表（进程记录）
  id              → 进程记录主键（game-start/stop/restart 用此 ID）
  server_type     → 1=登录服, 2=大厅服(main_server), 3=游戏服
  server_ref_id   → 关联 game_server.id（大厅服操作以此关联）
  manage_mode     → "manual"=只显示状态 / "local_command"=支持启停
  process_bin_name→ 进程可执行文件名（健康检查降级 + kill 名称）
  start_command   → 启动命令
  work_dir        → 启动工作目录
  http_health_url → 健康检查 URL（仅 host:port 生效）
```

## 5. Windows 配置指南

### 5.1 main_server env.toml（大厅服）

```toml
HttpUrl = "0.0.0.0:8090"  # GM HTTP 监听端口

# 鉴权（选一种）
GmToken = "your-strong-token-here"          # 推荐：Token 认证
# GmAllowIPs = ["127.0.0.1", "192.168.1.0/24"]  # 备选：IP 白名单
```

### 5.2 gm_server 中区服配置

在 admin 后台 **区服管理 → 编辑区服**，填写：
- `大厅服HTTP地址 (mainServerHttpUrl)`：填 `http://127.0.0.1:8090`（或实际 IP）

区服运行状态将自动 GET `http://127.0.0.1:8090/ping` 判断。

### 5.3 进程管理配置（Windows）

在 admin 后台 **进程管理 → 新增进程记录**：

| 字段 | 示例值 | 说明 |
|------|--------|------|
| 服务器类型 | 2 (大厅服) | |
| 关联区服ID | 1 | game_server.id |
| 进程名称 | 随意描述 | |
| 管理模式 | local_command | 支持启停 |
| 进程文件名 | main_server.exe | tasklist 查找用，也用于 kill |
| 启动命令 | `D:\servers\s1\main_server.exe` | 完整路径，支持参数 |
| 工作目录 | `D:\servers\s1` | |
| 健康检查URL | `http://127.0.0.1:8090` | host:port 即可 |

> **提示**：`启动命令`直接写可执行文件路径即可，gm_server 会包装为 `cmd /C <command>`。  
> 如需切换目录再启动，可写：`cd /d D:\servers\s1 && main_server.exe`

## 6. Linux 配置指南

### 6.1 main_server env.toml

```toml
HttpUrl = "0.0.0.0:8090"

GmToken = "your-strong-token-here"
# 或：
# GmAllowIPs = ["127.0.0.1", "10.0.0.0/8"]
```

### 6.2 进程管理配置（Linux）

| 字段 | 示例值 | 说明 |
|------|--------|------|
| 进程文件名 | `main_server` | pgrep -x 查找名，也用于 pkill |
| 启动命令 | `./main_server` 或 `nohup ./main_server &` | sh -c 包装执行 |
| 工作目录 | `/opt/game/s1` | |
| 健康检查URL | `http://127.0.0.1:8090` | |

> **提示**：gm_server 本身通过 `sh -c <command>` 启动进程，子进程是 gm_server 的子进程。  
> 若需持久化（gm_server 重启后进程不受影响），可在启动命令中使用：
> - `nohup ./main_server > run.log 2>&1 &`（简单场景）
> - `systemd` / `supervisor` 托管（推荐生产）；此时 `manage_mode` 改为 `manual`，仅监控不启停

### 6.3 使用 systemd（推荐）

```ini
# /etc/systemd/system/game-main-s1.service
[Unit]
Description=Game Main Server Zone 1
After=network.target

[Service]
Type=simple
User=gameuser
WorkingDirectory=/opt/game/s1
ExecStart=/opt/game/s1/main_server
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
systemctl enable game-main-s1
systemctl start game-main-s1
```

此时进程管理中 `manage_mode = "manual"`，`http_health_url = "http://127.0.0.1:8090"`，只做健康检查展示，不做启停。

## 7. GM HTTP 接口鉴权（X-GM-Token）

gm_server 转发操作到 main_server 时，需在请求头加入 Token：

在 gm_server 的转发逻辑中（`server_group.go` 的 `forwardToMainServer`），已有预留 header 传递，需确保转发 HTTP Client 带上 `X-GM-Token: <token>`。

若 gm_server 与 main_server 部署在**同一台机器**上，可用 IP 白名单代替 Token：

```toml
# main_server env.toml
GmToken = ""                    # 不填 Token
GmAllowIPs = ["127.0.0.1"]     # 只允许本机访问
```

## 9. 服务管理链路完整流程图

```
┌─────────────────────────────────────────────────────┐
│                  Admin 前端                          │
│                                                     │
│  区服状态监控页（30s 轮询）                           │
│  ┌──────────────────────┐ ┌──────────────────────┐  │
│  │  GET /api/servers/   │ │ GET /api/servers/    │  │
│  │    game-list         │ │   game-list-all      │  │
│  │（大厅服列表+状态）   │ │（游戏服进程列表+状态）│  │
│  └──────────┬───────────┘ └──────────┬───────────┘  │
└─────────────┼────────────────────────┼──────────────┘
              │                        │
              ▼                        ▼
┌─────────────────────────────────────────────────────┐
│                   gm_server                         │
│                                                     │
│  GmGetGameServerList              GmGetGameServerProcessList│
│  ┌────────────────────┐         ┌─────────────────────┐ │
│  │ 查 game_server 表  │         │查 server_process 表  │ │
│  │ (group_id > 0)     │         │(server_type = 3)     │ │
│  └─────────┬──────────┘         └──────────┬──────────┘ │
│            │                               │            │
│            ▼ mainServerHTTPReachable()     ▼ processHealthCheck()│
│  ┌─────────────────────┐    ┌──────────────────────────┐│
│  │ HTTP GET /ping      │    │ 1. TCP dial(http_health_url)│
│  │ (800ms 超时)        │    │ 2. tasklist/pgrep(bin_name)│
│  └─────────┬───────────┘    └──────────┬───────────────┘│
└────────────┼────────────────────────────┼────────────────┘
             │                            │
             ▼                            ▼
   main_server /ping             本地系统进程检测
   (HTTP 200 = 在线)             (进程存在 = 在线)


启停操作链路：
Admin [启动大厅] → POST /api/servers/start { serverId: game_server.id }
  → gm_server handleManagedServerLifecycle
  → 查 server_process WHERE server_ref_id = serverId
  → startProcess(): cmd /C <start_command> 或 sh -c <start_command>

Admin [启动游戏] → POST /api/servers/game-start { id: server_process.id }
  → gm_server handleProcessLifecycle
  → 查 server_process WHERE id = id
  → startProcess(): 同上
```
